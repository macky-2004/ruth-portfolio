Business Website

AVIKA Boutique

The Business Problem

AVIKA Boutique had a stunning physical store in Thanjavur but virtually no online presence. Customers couldn't view their fashion collections before visiting, see pricing or new arrivals, read testimonials from other customers, or book appointments online. They were losing potential customers to competitors with better digital visibility.

The Approach

I needed to create a digital experience that matched AVIKA's in-store luxury. This meant premium visual design with warm gold tones and elegant typography, fast loading (luxury shouldn't mean slow), mobile-first design, easy navigation, and social proof through testimonials.

What I Built

  • Responsive Mobile Navigation — Slide-out panel with smooth hamburger animation
  • Scroll-Adaptive Header — Transparent to frosted glass transition
  • Scroll Reveal Animations — Content fades in as visitors scroll
  • Custom Testimonial Carousel — Auto-advancing every 3 seconds, pauses on hover
  • Back-to-Top Button — Subtle floating button for easy navigation
  • SEO-Optimized Structure — Semantic HTML5, proper heading hierarchy, meta tags
  • Zero External Dependencies — Pure vanilla HTML, CSS, JavaScript

The Results

  • Website loads in under 2 seconds on mobile
  • Fully responsive across all devices
  • Improved local SEO visibility for "boutique Thanjavur" searches
  • Customers can now browse collections before visiting
  • Testimonials carousel increases trust and conversion

Portfolio / Creative

Madhavi Mehandi

The Business Problem

Madhavi was a talented mehendi artist with 50+ designs in her portfolio — but no way to show them to potential clients. She relied entirely on word-of-mouth and WhatsApp forwards of low-quality photos. No professional portfolio, no way to categorize designs, no booking system or clear pricing. Competitors with simple Instagram pages were winning clients.

The Approach

Madhavi needed more than a website — she needed a digital showcase that reflected her artistry. Key priorities: visual-first design, easy browsing with category filters, immersive viewing through a lightbox, fast loading for mobile brides, and a video element showing her process.

What I Built

  • Auto-Playing Hero Video — Shows Madhavi's mehendi process with image fallback
  • Interactive Gallery with Category Filters — Bridal, Casual, Festival, Arabic
  • Custom Lightbox Viewer — Full-screen with X, outside click, or Escape to close
  • Scroll Reveal Animations — Gallery items fade in dynamically
  • Scroll-Adaptive Navigation — Frosted glass header on scroll
  • BEM CSS Architecture — Organized, maintainable code
  • CSS clamp() Typography — Fluid responsive text sizing
  • 100% Vanilla Code — No external libraries, maximum performance

The Results

  • Professional portfolio that rivals competitors' Instagram presence
  • 50+ designs organized and easily browsable
  • Category filtering helps clients find exactly what they want
  • Lightbox creates immersive viewing experience
  • Mobile-optimized for brides browsing on their phones
  • 100% PageSpeed score — no bloated plugins

Business Website

TYM Jewellery

The Business Problem

TYM Jewellery had 14+ jewellery categories and 80+ products — but no organized way for customers to browse online. Customers couldn't browse categories without visiting the store, there was no system for online inquiries, no FAQ section (staff answered the same questions repeatedly), and no way to display customer testimonials. They needed a solution without complex e-commerce infrastructure.

The Approach

The goal wasn't to build an e-commerce site. It was to build a digital catalogue that made browsing effortless and inquiries frictionless. Strategy: category-based navigation, WhatsApp integration for instant communication, FAQ accordion, testimonials, and mobile-first design.

What I Built

  • Hidden Category Drawer — 14 master categories, 6 products each
  • WhatsApp API Integration — One tap to start a conversation
  • Infinite Feedback Slider — CSS keyframe-driven, pauses on hover/touch
  • FAQ Accordion — 5 common questions with smooth transitions
  • Care Policy Modal — Sliding overlay for jewellery care info
  • Golden Inline SVG Icons — Custom-designed, crisp at any resolution
  • Semantic HTML5 — Proper structure for screen readers and search engines
  • Serverless Architecture — 100% client-side, zero server costs

The Results

  • Customers can browse 80+ products in organized categories
  • WhatsApp inquiries increased customer engagement significantly
  • FAQ section reduced repetitive staff inquiries by ~40%
  • Zero server costs — hosted on free static platforms
  • Instant page loads improve user retention
  • Mobile-optimized for on-the-go jewellery browsing

Business Website

TYM Electricals & Hardware

The Business Problem

TYM Electricals & Hardware needed to modernize their digital presence without losing their established credibility. No comprehensive website showing their full product range, B2B customers couldn't find product specifications online, no lead generation system, needed multiple pages (Home, About, Products, Contact, Policies), and concerned about maintenance costs and complexity.

The Approach

A traditional business needs a modern website that still feels trustworthy. Strategy: professional corporate design (deep blue + warm gold), multi-page structure, WhatsApp lead generation, glassmorphism effects, and easy maintenance through static site architecture.

What I Built

  • Multi-Page Architecture — 7 pages with unique meta tags for SEO
  • Responsive Navigation — Desktop horizontal menu, mobile slide-out drawer
  • Scroll Animations — IntersectionObserver-powered fade-ins on every page
  • Custom Testimonial Carousel — Auto-rotation every 5 seconds, pause on hover
  • WhatsApp Lead Generation — Contact form redirects to WhatsApp
  • Glassmorphism Header — backdrop-filter: blur(20px)
  • Responsive Grid Layouts — Product showcases, service cards, stats grids
  • Inline SVG Icons — Custom icons, CSS-driven color changes
  • Zero External Dependencies — Pure vanilla code

The Results

  • 7-page comprehensive site improves SEO coverage significantly
  • WhatsApp integration generates direct customer inquiries daily
  • Glassmorphism header creates memorable first impression
  • Multi-page structure allows detailed product explanations
  • Fast loading on all devices improves user retention
  • Static architecture means zero ongoing server costs

College Project

Web Portal for Property Management System

The Business Problem

Property management is a universal challenge: landlords struggle to list vacancies, tenants can't find reliable rentals, and both parties need a system to manage bookings and communications. For my college capstone project, I chose to solve this real-world problem rather than build something theoretical.

The Approach

Build a full-stack web application that simplifies the rental process for all stakeholders: landlords need to list properties and manage inquiries, tenants need to search/filter and book viewings, admins need oversight of all platform activity, and everyone needs a responsive, easy-to-use interface.

What I Built

  • User Authentication — Secure login/registration with password hashing
  • Property Listings — Detailed pages with images, pricing, location, amenities
  • Search & Filter — Find properties by location, price range, type, amenities
  • Booking Management — Schedule viewings, landlord approve/reject
  • Admin Dashboard — Overview of users, properties, bookings, analytics
  • Responsive UI — Works flawlessly on mobile and desktop
  • Database-Driven — MySQL backend with normalized tables

The Results

  • Demonstrated full-stack development capabilities
  • Applied database normalization and relational design principles
  • Implemented secure authentication and authorization workflows
  • Created responsive UI that works across all devices
  • Received positive academic evaluation for real-world applicability

This project was developed as a college assignment and the source code is private. I'm happy to discuss the architecture and demonstrate similar functionality for your project.
